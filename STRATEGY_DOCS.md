```
    │   └── {ticker}_inst.csv
    └── tdcc/                       集保股權分散表
        └── {ticker}_tdcc.csv
```

### 插件架構

每個策略繼承 `BaseStrategy`，實作三個方法：

| 方法 | 功能 |
|------|------|
| `get_params()` | 定義可調參數，Streamlit 自動生成滑桿 |
| `generate_signals(df, params)` | 訊號邏輯，輸出 `signal / stop_loss / state` |
| `get_audit_config()` | 稽核設定（跑哪些項目、猴子次數等）|

新增策略只需在 `strategies/` 放入新 `.py` 檔，重啟 APP 自動偵測。

## 2. 策略一：Stage 2 突破（變盤三部曲）

**檔案**：`strategies/stage2.py`　　**版本**：v1.1

**靈感來源**：詩魂頻道《Stan Weinstein 四階段理論 × 裸K版》

### 核心思想

超過 95% 的超級飆股漲幅發生在 Stage 2（上升趨勢）。策略目標是在趨勢剛從下跌轉為上漲的初期確認後進場，搭主力便車。

### 策略步驟（SOP）

**口訣：HH+HL 越來越高 = 上升趨勢；LH+LL 越來越低 = 下跌趨勢**

#### 第一步：確認變盤三部曲

| 部曲 | 條件 | 若不成立 |
|------|------|---------|
| 一部曲 | 收盤突破「末跌高」（最後一個 Lower High）| 絕對不碰 |
| 二部曲 | 突破後回測，不可再破底（HL 守住）| 趨勢回到下跌，放棄 |
| 三部曲 | 再次突破前高（破平台 BOS，形成 HH）| 等待 |

#### 第二步：三重確認（缺一不可）

| 重 | 條件 |
|----|------|
| 第一重 | 趨勢結構：完成三部曲（BOS 突破）|
| 第二重 | RS 相對強度 > RS 均線（個股強於大盤）|
| 第三重 | 成交量放大（> 20 日均量 × 1.1）|

#### 第三步：進場

- 三部曲 + 三重確認同時成立 → `signal = "buy"`
- 停損 = 收盤價 − ATR × `atr_mult`

#### 第四步：持倉管理

- 獲利超過 1:1 風險報酬後，停損上移（移動停損）
- 每次再突破平台，停損跟上

#### 第五步：出場

- 出現「翻多為空」（Higher Low 被跌破）→ 全數平倉
- 停損被觸及 → 平倉

### 可調參數

| 參數 | 預設 | 說明 |
|------|------|------|
| `swing_n` | 10 | 高低點判定回溯 K 棒數（越大 = 越大級別）|
| `atr_mult` | 1.5 | 停損寬鬆度（ATR 倍數）|
| `vol_confirm` | 1.1 | 量能確認門檻（相對均量）|

### 訊號欄位

| 欄位 | 說明 |
|------|------|
| `signal` | `buy` / `sell` / `hold` |
| `stop_loss` | 當根 K 棒的停損價 |
| `state` | `stage2` / `basing` / `stage1` / `downtrend` / `stage3` |
| `entry_reason` | 進場原因說明 |
| `exit_reason` | 出場原因說明 |

### 策略示意

```
下跌段    →    末跌高    →    打底（不破底）    →    破平台+三重確認 → 進場
LH+LL         突破                HL守住                BOS+RS+量
(downtrend)  (stage1)           (basing)              (stage2=BUY)
```

## 3. 策略二：A+B+C 籌碼趨勢（M哥完整版）

**檔案**：`strategies/abc_institutional.py`　　**版本**：v3.0

**靈感來源**：M哥方格子系列文章（四篇）

| 文章 | 核心貢獻 |
|------|---------|
| A+B+C 核心架構 | 趨勢×籌碼×時間三維框架 |
| 光洋科時間尺度 | 慣性改變、大時間尺度觀察 |
| 中美晶完整評估 | X型態、外資斜率加速、量縮小平台、多時間框架 |
| 股東結構質變 | 股東大增+散戶持股減少+中實戶增加 = 籌碼內部質變 |

### 核心口訣

**趨勢很重要、籌碼先知道、時間是金子**

### A：趨勢條件

| 版本 | 條件 | 用途 |
|------|------|------|
| `A_full` | MA240 斜率 > 0 且站上一年新高 且 多頭排列 | 標準進場 |
| `A_flat` | MA240 斜率 ≥ -0.2%（均線走平）且短均 > 長均 | XB 型態進場 |

**240 日扣抵邏輯**：未來扣抵值若在低位，只要股價維持不變，均線即將被拉平往上，是「滿壘等強迫進壘」的狀態。

### X 型態（比 A 更早的前驅訊號）

**定義**：空頭環境中，突然出現大量且底部不再破底（底部越墊越高）

```
條件：
  ① MA_long_slope < 0.5%（長均尚未明顯上翹）
  ② 近 x_hl_days 天出現大量（> 均量 × x_vol_mult）
  ③ 近期低點 > N天前低點（HL 遞增）
  ④ 尚未站上一年新高（尚非 A_full）
```

用途：X 型態 + B1 籌碼 → `PRE` 預警訊號，提前加入觀察清單。

### B1：外資籌碼三層確認

| 層級 | 條件 | 文章依據 |
|------|------|---------|
| `B1a` 基礎 | 近 N 日累積淨買超 ≥ 門檻（張數）| A+B+C 原文 |
| `B1b` 斜率加速 | 最近累積淨買 > N天前（斜率在上升）| 中美晶：外資斜率越大越積極 |
| `B1c` 慣性改變 | 股價下跌時外資仍淨買，近10日≥3次 | 光洋科：最重要的買進訊號 |

**B1 成立條件**：`B1a` 必須 + (`B1b` 或 `B1c`) 任一

> 無三大法人資料時：自動用成交量動能替代（`fi_net = (量/均量 - 1) × 1000`），`inst_source` 欄位顯示「proxy」。

### B2：量縮 ATR 收縮小平台

**文章描述**：「藍色框框是一個又一個量縮價穩的小平台，是我最喜歡觀察的地方。每一個小平台都是一道防線。」

| 子條件 | 計算方式 |
|--------|---------|
| 量縮 | 近 N 日均量 < 20日均量 × `vol_shrink` |
| ATR 收縮 | 近 N 日 ATR 均值 < 長期 ATR 均值 × `atr_shrink`（波動收窄）|
| 價穩 | 近 N 日最低點 ≥ 前一期最低點（不破底）|

三項同時成立 → `B2_ok = True`

### 股東結構質變（文章4，加分項）

**資料來源**：集保結算所週報（`fetch_tdcc.py` 抓取）

| 訊號 | 條件 | 意義 |
|------|------|------|
| `quality_change` | 總股東人數↑ + 散戶持股比例↓ + 中實戶持股比例↑ | 籌碼內部結構質變，聰明錢在接貨 |
| `smart_money_in` | 中間段（50-400張）人數與持股比例同步上升，連續2週 | 中實戶規模性進場 |
| `chips_clean` | 散戶持股比例低於4週均線（籌碼越來越乾淨）| 套牢散戶陸續離場 |

> 無集保資料時：三個訊號全設為 False，BUY 訊號仍可觸發（加分項，無資料不扣分）。

### 融資方向輔助過濾

**文章描述**：「融資早進早離是散戶沒有企圖心；外資之後進場才是真正力量。融資在減少 + 外資在增加 = 籌碼交換進行中。」

- 若有融資資料：要求 `margin_net < 0`（融資在減少）才允許進場
- 無資料：不限制

### 訊號三級體系

```
觀察清單         等待區              進場
PRE          →  SETUP          →   BUY
X型態+B1       A/Aflat+B1         A+B1+B2+月線+融資(+股東質變)
最早預警        等量縮平台            完整訊號
```

| 等級 | 建議行動 |
|------|---------|
| `PRE` | 加入觀察清單，開始追蹤籌碼變化 |
| `SETUP` | 準備好資金，等 B2 量縮平台出現 |
| `BUY` | 評估後進場，依停損設定部位大小 |

### 可調參數

| 參數 | 預設 | 說明 |
|------|------|------|
| `ma_long` | 240 | 長期均線週期 |
| `ma_mid` | 60 | 中期均線週期 |
| `ma_short` | 20 | 短期均線週期 |
| `high_period` | 252 | 一年新高判斷週期 |
| `x_hl_days` | 20 | X 型態底部墊高確認天數 |
| `x_vol_mult` | 1.5 | X 型態大量倍數門檻 |
| `inst_days` | 10 | 外資累積淨買天數 |
| `inst_thresh` | 3000 | 外資累積淨買張數門檻 |
| `slope_accel` | 5 | 斜率加速確認天數 |
| `platform_days` | 5 | 量縮小平台最少天數 |
| `vol_shrink` | 0.75 | 量縮門檻 |
| `atr_shrink` | 0.8 | ATR 收縮門檻 |
| `use_tdcc` | 自動 | 股東結構過濾模式 |
| `atr_stop` | 2.0 | 停損 ATR 倍數 |

### 出場條件

| 條件 | 說明 |
|------|------|
| `MA_long_slope < 0` 連續 3 日 | 長期均線轉下，趨勢惡化 |
| 外資累積淨賣超過門檻 | 籌碼大量流出 |
| 停損被觸及 | `close ≤ stop_loss` |

## 4. 共用回測引擎規格

**檔案**：`app.py` 中的 `run_backtest()`

### 資金管理

| 項目 | 設定 |
|------|------|
| 初始資金 | 100 萬台幣 |
| 每筆最大風險 | 資金 × 2%（`POSITION_RISK = 0.02`）|
| 部位計算 | `shares = floor(最大損失 / 風險 / 1000) × 1000`（千股為單位）|
| 最小部位 | 1000 股（1 張）|

### 交易成本

| 項目 | 說明 |
|------|------|
| 手續費 | 單邊費率（可在 APP 側欄調整，預設 0.1%）|
| 滑點 | 單邊滑點率（可調整，預設 0.1%）|
| 進場成本 | 進場價 = 收盤 × (1 + 手續費 + 滑點) |
| 出場收益 | 出場價 = 收盤 × (1 - 手續費 - 滑點) |

### 移動停損

- 獲利達到 1:1 風險報酬後啟動移動停損
- 新停損 = max(舊停損, 當前收盤 − ATR × atr_mult)

### 績效統計欄位

| 指標 | 計算方式 |
|------|---------|
| 總報酬率 | (期末資產 − 初始資金) / 初始資金 |
| 年化報酬 | (期末/期初)^(365/天數) − 1 |
| 最大回撤 | (期間最高點 − 當日) / 最高點 的最大值 |
| 夏普比率 | 日報酬均值 / 標準差 × √252 |
| 勝率 | 獲利筆數 / 總筆數 |
| 獲利因子 | 總獲利 / 總虧損絕對值 |

## 5. 稽核系統說明

**目的**：驗證策略是否真實有效，而非資料過擬合或運氣好。

### A. 策略 vs 買入持有

比較策略均報酬率與相同期間買入持有的報酬率。

- **通過**：策略均報酬 > 買入持有均報酬

### B. 參數敏感度熱圖

對策略前兩個可調參數，各取 ±1 步長，共 3×3 = 9 組參數跑回測，觀察報酬率分散程度。

- **通過**：最高與最低報酬差距 < 30%
- **警告**：30% ≤ 差距 < 50%
- **未通過**：差距 ≥ 50%（高度參數依賴）

### C. 分段績效（牛 / 震盪 / 熊）

將回測期間三等分，分別計算策略與買入持有在各段的報酬率。

- **通過**：熊市段策略報酬 > 買入持有報酬（有防禦效果）

### D. 猴子亂買對照

300 次隨機進出場模擬，與策略均報酬比較。

- **通過**：少於 40% 的猴子能打敗策略

### E. 交易成本衝擊

在零成本、低成本（0.1%+0.1%）、一般（0.2%+0.2%）、高成本（0.3%+0.3%）四個情境下各跑一次。

- **通過**：≥ 3 個情境仍為正報酬
- **警告**：2 個情境正報酬

### F. Walk-Forward 測試（過擬合檢測）

將資料對半切，前半訓練、後半樣本外測試，對每檔股票各做一次。

- **通過**：樣本外正報酬比例 > 50% 且平均衰退 < 20%
- **警告**：正報酬 > 50% 但平均衰退 ≥ 20%

## 6. 資料來源與更新流程

### 資料來源

| 資料 | 來源 | 頻率 | 爬蟲檔案 |
|------|------|------|---------|
| 個股 OHLCV | Yahoo Finance | 每日 | `download_tw_stocks.py` / `updater.py` |
| 大盤指數 | Yahoo Finance `^TWII` | 每日 | 同上 |

| 集保股權分散表 | 集保結算所 | 每週四 | `fetch_tdcc.py` |

### 每日排程（`run_daily.py`）

```
18:30 收盤後
  Step 1 → updater.py          更新個股股價（增量，只抓近 7 天）
  Step 2 → fetch_institutional  更新三大法人（近 7 天）
  Step 3 → 掃描所有策略訊號
  Step 4 → notifier.py          推播 Line / Email
```

**設定排程**（Mac / Linux）：

```bash
crontab -e
# 加入：
30 18 * * 1-5 cd /你的路徑 && python run_daily.py >> logs/daily.log 2>&1
```

**集保資料為週頻，建議每週四另外執行：**

```bash
# crontab 加入：
0 20 * * 4 cd /你的路徑 && python fetch_tdcc.py >> logs/tdcc.log 2>&1
```

### 首次設定流程

```bash
# 1. 安裝套件
pip install streamlit yfinance pandas numpy matplotlib requests

# 2. 下載股價歷史資料
python download_tw_stocks.py

# 3. 下載三大法人歷史資料（需時較長）
python fetch_institutional.py --mode history --start 2020-01-01

# 4. 下載集保股權歷史資料（需時最長，每週一筆）
python fetch_tdcc.py --mode history --start 2020-01-01

# 5. 設定推播（編輯 config.json）
# 6. 啟動 APP
streamlit run app.py
```

## 7. 如何新增策略

只需三步，不需修改任何現有檔案：

```python
# strategies/my_strategy.py

from strategies.base import BaseStrategy
import pandas as pd
import numpy as np

class MyStrategy(BaseStrategy):
    name        = "我的策略名稱"       # APP 側欄自動顯示
    description = "一句話說明策略邏輯"
    version     = "1.0"

    def get_params(self) -> dict:
        # 定義可調參數 → Streamlit 自動生成對應的滑桿/選單
        return {
            "period": {
                "type": "int",          # int / float / select
                "default": 20,
                "min": 5, "max": 60, "step": 1,
                "label": "週期",
            },
        }

    def generate_signals(self, df: pd.DataFrame, params: dict) -> pd.DataFrame:
        df = df.copy()
        period = int(params.get("period", 20))

        # df 已含：Open, High, Low, Close, Volume, ATR, Vol_MA, RS, RS_MA
        # 你只需要填入以下三個欄位：
        df["signal"]    = "hold"          # "buy" / "sell" / "hold"
        df["stop_loss"] = float("nan")    # 停損價
        df["state"]     = "unknown"       # 任意字串，顯示在訊號面板

        # 寫你的邏輯 ...
        ma = df["Close"].rolling(period).mean()
        df.loc[df["Close"] > ma, "signal"] = "buy"
        df.loc[df["Close"] > ma, "stop_loss"] = ma

        return df

    def get_audit_config(self) -> dict:
        return {
            "tests":     ["A", "B", "C", "D", "E", "F"],
            "benchmark": "buy_hold",
            "monkey_n":  300,
            "wf_split":  0.5,
        }
```

1. 存入 `strategies/my_strategy.py`
2. 重啟 `streamlit run app.py`
3. 側欄策略選單自動出現新策略，六項稽核自動運行 ✅

## 8. 已知限制與注意事項

### 資料限制

| 限制 | 說明 |
|------|------|
| Yahoo Finance 穩定性 | 偶爾連線失敗，`updater.py` 內建重試機制 |
| 三大法人 API | 台灣證交所非正式 API，格式偶有變動，需維護 `fetch_institutional.py` |
| 集保資料頻率 | 每週一次，日線策略只能以前一週數值 forward fill |
| 融資資料 | 目前三大法人 API 未包含融資明細，`margin_net` 欄位需另外來源 |

### 策略限制
